# IMU 姿态解算算法优化报告

## 优化目标
基于真实标注数据（4671条静态姿态记录）优化动物姿态识别算法，解决以下问题：
1. 站立和趴卧容易混淆
2. 仰卧和趴卧被误判为侧躺
3. 站立摇头被误判为行走
4. 姿态转换过程被误判为运动

## 数据分析

### 关键发现
- **站立 vs 趴卧**：roll几乎相同（113.3° vs 113.4°），只能通过pitch区分
  - 站立：pitch = 16.1 ± 5.8°，范围[-0.1, 29.1]
  - 趴卧：pitch = 10.2 ± 2.2°，范围[5.5, 17.2]
  - 重叠区域：[5.5, 17.2]°

- **站立摇头**：17%的站立数据角速度 > 15°/s，5.7% > 20°/s

- **最佳阈值**：通过平衡分数分析，13.5°是最佳阈值
  - 趴卧准确率：92.3%
  - 站立准确率：75.2%
  - 平衡分数：82.9（最高）

## 算法优化

### 1. 静态姿态识别（规则优先级）

```javascript
// 规则1：右侧躺（roll < -157°）
// 规则2：左侧躺（16° < roll < 29°）
// 规则3：仰卧（pitch > 30°）
// 规则4：低头进食（pitch < -10° 且 87° < roll < 112°）
// 规则5：趴卧 vs 站立（108° < roll < 119°）
//   - pitch < 13.5° → 趴卧
//   - pitch >= 13.5° → 站立
// 规则6：其他情况使用加权距离匹配
```

**优先级设计原因**：
- 侧躺的roll最独特，优先判断
- 仰卧的pitch最独特（>30°），次之
- 趴卧和站立最后区分（roll相同，只能靠pitch）

### 2. 运动状态判断

```javascript
// 奔跑：角速度 > 23°/s 且加速度 > 1.1g
// 行走：角速度 > 20°/s 且加速度方差 > 0.02
```

**更严格的阈值**：避免站立摇头被误判为行走

### 3. 状态转换保护

```javascript
// 静态姿态 → 运动状态需要：
// 1. 持续高角速度至少1.5秒
// 2. 角速度 > 25°/s
```

**保护机制**：
- 过滤站立摇头（短暂高角速度）
- 过滤姿态转换（翻身等）
- 确保只有真正的行走/奔跑才被识别

### 特殊场景处理

✅ **仰卧/趴卧不会被误判为侧躺**
- 规则优先级保证侧躺优先判断
- 测试结果：0次误判

✅ **站立摇头不会被误判为行走**
- 需要持续1.5秒高角速度
- 角速度阈值提高到20°/s

✅ **姿态转换不会被误判为运动**
- 需要角速度 > 25°/s
- 过滤翻身等短暂高角速度
